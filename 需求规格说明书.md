# 需求规格说明书：中国海外文物知识图谱系统

## 1. 引言

### 1.1 编写目的

本需求规格说明书旨在明确中国海外文物知识图谱系统的功能需求、非功能需求和约束条件，为系统设计、开发、测试和验收提供依据。预期读者包括项目开发人员、测试人员、指导教师和评审专家。

### 1.2 项目背景

中国大量珍贵文物流散于海外博物馆，这些文物信息分散在各个博物馆的独立数据库中，缺乏统一的语义化关联和知识组织。本系统通过采集海外博物馆的中国文物数据，构建知识图谱，实现文物信息的语义化整合与关联查询。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 知识图谱 | 以图结构组织的知识库，由节点（实体）和边（关系）构成 |
| 三元组 | 知识图谱的基本单元，格式为（头实体，关系，尾实体） |
| MERGE | Neo4j Cypher 语句，若节点存在则匹配，不存在则创建 |
| CORS | 跨源资源共享，浏览器安全策略限制不同域之间的资源访问 |
| accession_number | 馆藏编号，博物馆分配给每件藏品的唯一标识符 |
| 断点续爬 | 爬虫中断后可从上次位置继续，避免重复采集 |

## 2. 系统概述

### 2.1 系统目标

构建一个中国海外文物知识图谱系统，实现以下目标：

1. 自动采集三家美国博物馆的中国文物数据（目标总量 7000+ 件）
2. 对原始数据进行清洗、翻译和标准化处理
3. 将结构化数据导入关系数据库进行持久化存储
4. 基于关系数据库构建知识图谱，支持语义化关联查询
5. 解决图片跨域访问问题，支持云端部署

### 2.2 系统边界

**系统范围内：**

- 数据采集（三家博物馆）
- 数据清洗与翻译
- 关系数据库存储（MySQL）
- 知识图谱构建（Neo4j）
- 图片 URL 代理
- 容器化环境编排

**系统范围外：**

- 前端可视化界面
- 用户注册与认证
- 自然语言问答
- 移动端应用

### 2.3 用户角色

| 角色 | 描述 | 使用方式 |
|------|------|---------|
| 数据工程师 | 负责数据采集、清洗和图谱构建 | 运行 Python 脚本，执行命令行工具 |
| 知识图谱研究员 | 查询和分析文物知识图谱 | 通过 Neo4j Browser 执行 Cypher 查询 |
| 系统管理员 | 部署和维护系统运行环境 | 管理 Docker 容器，配置云服务器 |
